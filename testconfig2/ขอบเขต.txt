2.3.6 ระบบจัดการเอกสาร (Documents Module) ถูกออกแบบเพื่อรองรับการจัดเก็บไฟล์ที่เกี่ยวข้องกับการดำเนินงานด้านระบบเครือข่าย ทั้งไฟล์ค่าตั้งค่า (Config Files) เอกสารประกอบ ไดอะแกรม และไฟล์อื่น ๆ ที่เกี่ยวข้อง โดยมีขอบเขตความสามารถดังต่อไปนี้
2.3.6.1 ระบบสามารถจัดการไฟล์ภายในโปรเจกต์ได้ดังนี้ 
2.3.6.1.1 สร้างโฟลเดอร์ใหม่ (New Folder) เพื่อแบ่งหมวดหมู่ไฟล์
2.3.6.1.2 เปลี่ยนชื่อโฟลเดอร์และไฟล์ (Rename) เพื่อให้สอดคล้องกับเอกสาร
2.3.6.1.3 ลบโฟลเดอร์และไฟล์ (Delete) สำหรับไฟล์หรือโฟลเดอร์ที่ไม่ต้องการ
2.3.6.1.4 จัดระเบียบไฟล์ (File Organization) ผู้ใช้สามารถย้ายไฟล์เข้าสู่โฟลเดอร์
2.3.6.1.5 ไม่สามารถเปลี่ยนชื่อ ย้าย หรือลบไฟล์ตั้งค่าอุปกรณ์ในโฟลเดอร์ Config ได้ สามารถทำได้เพียงเปิดดูข้อมูลเท่านั้น
2.3.6.2 ระบบสามารถแสดงตัวอย่างไฟล์แบบทันทีสำหรับไฟล์ต่อไปนี้ 
2.3.6.2.1 ภาพ (Images) ได้แก่ png, jpg, jpeg
2.3.6.2.2 ไฟล์เอกสาร ได้แก่ PDF
2.3.6.2.3 ไฟล์ข้อความ ได้แก่ .txt 
2.3.7 ระบบมีหน้าจอ Command Templates สำหรับแนะนำชุดคำสั่งในการดึงค่าตั้งค่าจากอุปกรณ์เครือข่ายแต่ละยี่ห้อ เพื่อให้ผู้ใช้งานสามารถรวบรวมข้อมูลได้ครบถ้วนตามที่ระบบต้องการ โดยชุดคำสั่งที่แนะนำจะครอบคลุมข้อมูลตามหัวข้อ 2.3.2
2.3.8 ระบบรองรับการจัดเก็บและแสดงประวัติการอัปโหลดไฟล์ทุกประเภท (Configuration Files และ Document Files) เพื่อให้สามารถตรวจสอบย้อนหลัง ติดตามการเปลี่ยนแปลง (Change Tracking) และเสริมความโปร่งใสในการบริหารจัดการระบบเครือข่าย โดยมีความสามารถดังนี้
2.3.8.1 Upload Logging and Metadata Tracking ทุกการอัปโหลดไฟล์ ระบบจะบันทึก Metadata ครบถ้วนในรูปแบบเดียวกัน ประกอบด้วย:
2.3.8.1.1 Time วันเวลาที่อัปโหลดไฟล์
2.3.8.1.2 Responsible User ผู้ใช้งานที่เป็นผู้ดำเนินการอัปโหลด
2.3.8.1.3 Activity Type  ประเภทเหตุการณ์
2.3.8.1.4 Site  ตำแหน่งของอุปกรณ์หรือเอกสาร
2.3.8.1.5 Operational Timing  ช่วงเวลาของเหตุการณ์
2.3.8.1.6 Purpose วัตถุประสงค์
2.3.8.1.7 Description  รายละเอียดการอัปโหลด
2.3.8.2 การแสดงประวัติแบบตาราง (History Table View) ระบบแสดงประวัติการอัปโหลดในรูปแบบตาราง แยกเป็น ประวัติการอัปโหลด Config  ประวัติการอัปโหลด Document  ประวัติรวม (Log Updates) และ ประวัติสำรองต่ออุปกรณ์ (Per-Device Backup History) โดยตารางประกอบด้วยคอลัมน์
2.3.8.2.1 Time วันเวลาที่อัปโหลดไฟล์
2.3.8.2.2 Responsible User ผู้ใช้งานที่เป็นผู้ดำเนินการอัปโหลด
2.3.8.2.3 Activity Type  ประเภทเหตุการณ์
2.3.8.2.4 Site  ตำแหน่งของอุปกรณ์หรือเอกสาร
2.3.8.2.5 Operational Timing  ช่วงเวลาของเหตุการณ์
2.3.8.2.6 Purpose วัตถุประสงค์
2.3.8.2.7 Description  รายละเอียดการอัปโหลด

2.3.8.3 การดาวน์โหลดไฟล์ย้อนหลัง (File Retrieval) ผู้ใช้สามารถดาวน์โหลดไฟล์ทุกไฟล์ที่เคยอัปโหลดแล้วแสดงในประวัติ
2.3.8.4 การส่งออกข้อมูลประวัติ (History Export) ระบบรองรับการส่งออก ประวัติรวม (Log Updates) ทั้งหมดในรูปแบบ CSV เพื่อใช้ในการตรวจสอบหรือวิเคราะห์เพิ่มเติม
2.3.9 ระบบรองรับการกำหนดสิทธิ์ตามบทบาทของผู้ใช้งาน (Role) เพื่อควบคุมการเข้าถึงและการจัดการโปรเจคอย่างเป็นระบบ ผู้ใช้แต่ละคนจะสามารถเข้าถึงและจัดการข้อมูลได้เฉพาะภายในโปรเจคที่ตนเป็นสมาชิกและตามสิทธิ์ที่ได้รับมอบหมายเท่านั้น โครงสร้างสิทธิ์ถูกออกแบบให้แยกหน้าที่และความรับผิดชอบของแต่ละบทบาทอย่างชัดเจน เพื่อให้การทำงานร่วมกันมีความปลอดภัย โปร่งใส และตรวจสอบได้
2.3.9.1 Role Permission  การกำหนดสิทธิ์ตามบทบาทผู้ใช้งาน ระบบรองรับผู้ใช้งานภายในโปรเจคจำนวน 4 ระดับสิทธิ์ ได้แก่ Admin, Manager, Operator, และ Viewer โดยกำหนดสิทธิ์การเข้าถึงและการจัดการแตกต่างกันตามระดับบทบาท ดังนี้

2.3.9.1.1.2 ดูรายละเอียดโปรเจคทั้งหมดได้
